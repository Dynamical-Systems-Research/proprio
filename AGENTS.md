# Proprio repository contract

- Public and code-facing observability names are `procedural`, `validity`, and `support`.
- Do not expose numbered internal phase names in modules, classes, schema fields, CLI commands,
  artifacts, tests, or documentation.
- Name self-verifying acquisition by function (`skill drafting`, `skill admission`), not by an
  internal project sequence number.
- DSV4 prompts must state source precedence, exact output shape, forbidden behaviors, and an
  explicit preflight checklist. Preserve `reasoning_content` in raw assistant messages.
- Model self-judgment is evidence, never skill-admission authority.
