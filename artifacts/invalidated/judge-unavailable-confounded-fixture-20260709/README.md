# Invalidated unavailable-evidence fixtures

These four cases are excluded from judge-metrology metrics. The intended variable was an
unavailable simulator link, but the fixture also supplied a speculative code edit and an
unsupported diagnosis. `REJECT` was therefore defensible and conflicted with the expected
honest `HOLD`. The corrected fixture isolates unavailability: initial and final skills are
identical, no repair is submitted, and no independent defect is injected.
